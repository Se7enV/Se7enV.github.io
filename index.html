1.1算法和数据结构

算法（algorithm）

衡量算法好坏的重要标准

  • 时间复杂度
  • 空间复杂度

运用领域

  1. 运算
  2. 查找
  3. 排序
  4. 最优决策
  5. 面试（hhhhhh）

数据结构（data structure）

是数据的组织、管理和存储格式，其使用目的是为了高效地访问和修改数据。

组成方式

  1. 线性结构

    最简单的数据结构，包括数组、链表，以及由它们衍生出来的栈、队列、哈希表。

  2. 树

    相对复杂的数据结构，比较有代表性的是二叉树，由它又衍生出了二叉堆之类的数据结构。

  3. 图

    更为复杂的数据结构，因为在图中会呈现出多对多的关联关系。

  4. 其他数据结构

    都是由基本数据结构变形而来，用于解决某些特定问题，如跳表、哈希链表、位图等。

算法和数据结构的关系

有了数据结构这个舞台，算法才可以尽情舞蹈。

在解决问题时，不同的算法会选用不同的数据结构。

1.2时间复杂度

基本操作执行次数

场景1

1个长度10cm的面包，每3分钟吃掉1cm，那么吃掉整个面包需要多久？

3 x 10 = 30分钟

如果面包的长度是n cm，此时需要 3 x n = 3n分钟。

此时用一个函数表达吃掉整个面包所需要的时间，T(n) = 3n，n为面包的长度。

场景2

1个长度16cm的面包，每5分钟吃掉面包剩余长度的一半，即第5分钟吃掉8cm，第10分钟吃掉4cm，第15分钟吃掉2cm，那么把面包吃得只剩1cm，需要多久？

用数学方式表达就是，数字16不断地除于2，除几次以后的结果等于1？即以2为底16的对数$log_2 16$

（后续的对数函数的底数全部省略）

因此，把面包吃得只剩下1cm，需要 5 x log16 = 20分钟。

如果面包的长度是n cm，此时需要5乘以logn = 5logn分钟，记作T(n) = 5logn

场景3

1个长度10cm的面包和1个鸡腿，每2分钟吃掉1个鸡腿。那么吃掉整个鸡腿需要多久？

答案自然是2分钟。因为这里只要求吃掉鸡腿，和10cm的面包没有关系。

如果面包的长度是n cm呢？

无论面包多长，吃掉鸡腿的时间都是2分钟，记作T(n) = 2

场景4

1个长度为10cm的面包，小灰吃掉第1个1cm需要1分钟，吃掉第2个1cm需要2分钟，吃掉第3个1cm需要3分钟时间……每吃1cm所花的时间就比吃上一个多用1分钟。那么吃掉整个鸡腿需要多久？

答案是从1累加到10的总和，也就是55分钟。

如果面包的长度是n cm呢？

根据高斯算法，此时吃掉整个面包需要 1+2+3+…+(n-1)+n 即 (1+n) x n/2分钟，也就是$0.5n^2+0.5n$分钟，

记作$T(n) = 0.5n^2+0.5n$

程序基本操作执行次数的统计

设T(n)为程序基本操作执行次数的函数（也可以认为是程序的相对执行时间函数），n为输入规模，刚才的4个场景分别对应了程序中最常见的4种执行方式。

场景1

T(n) = 3n，执行次数是线性的。

场景2

T(n) = 5logn，执行次数是用对数计算的。

场景3

T(n) = 2，执行次数是常量。
